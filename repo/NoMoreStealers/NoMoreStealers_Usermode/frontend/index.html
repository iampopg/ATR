Enable Test Signing

First, we need to enable Test Signing mode on your system. This allows unsigned drivers to load.

Enable Test Signing (or disable Secure Boot temporarily if necessary) and temporarily disable antivirus.

bcdedit /set testsigning on

Important: Reboot your computer after running this command. If you cannot enable test signing via BCDEdit, enter BIOS/UEFI during boot (DEL/F2) and temporarily disable Secure Boot.

Build the Driver

Now let's build the driver in Visual Studio.

Open the project in Visual Studio and:

  • All driver signing configuration removed
  • Simply build in Release x64 mode
  • Driver will compile as unsigned binary

Note: If you still get signing errors, right-click the project → Properties → Driver Signing → Sign Mode: Off

Copy Driver to System Directory

Copy the compiled driver file to the Windows system drivers folder.

Copy the driver file to the Windows system drivers directory:

C:\Windows\System32\drivers\NoMoreStealers.sys

Tip: You'll need administrator privileges to copy files to this directory.

Configure Minifilter Instance

Configure the registry to register the minifilter instance.

Run these commands in an elevated Command Prompt (run as Administrator):

reg add "HKLM\SYSTEM\CurrentControlSet\Services\NoMoreStealers\Instances" /v DefaultInstance /t REG_SZ /d "NoMoreStealers Instance" /f reg add "HKLM\SYSTEM\CurrentControlSet\Services\NoMoreStealers\Instances\NoMoreStealers Instance" /v Altitude /t REG_SZ /d "370000" /f reg add "HKLM\SYSTEM\CurrentControlSet\Services\NoMoreStealers\Instances\NoMoreStealers Instance" /v Flags /t REG_DWORD /d 0 /f

Load the Driver

Almost there! Let's load the driver into the system.

Load the driver using the Filter Manager Control command in an elevated Command Prompt:

fltmc load NoMoreStealers

If successful, the driver is now loaded and the protection system should be active!

Unload the Driver (Optional)

When you're done using NoMoreStealers, you can unload the driver.

To unload the driver when you're finished:

fltmc unload NoMoreStealers

All Done! After completing all steps, restart the NoMoreStealers application. The driver status will be automatically detected and the protection system will activate.

Anti Rat Killswitch

Advanced protection against Remote Access Trojans

Killswitch Protection

When enabled, the killswitch monitors all processes for active TCP connections. Any unsigned process with an active network connection will be automatically terminated, protecting you from Remote Access Trojans (RATs) and other malware.

Killswitch Active

Warning

Use with caution: This feature will terminate any unsigned process with an active TCP connection. This may affect legitimate applications that are unsigned. Make sure you understand the implications before enabling.

Protecting users since day one • Fighting the good fight • No profit, just protection

What It Does

NoMoreStealers is a kernel-level minifilter driver that monitors and blocks unauthorized access to sensitive files and processes. It specifically targets information-stealing malware that attempts to access browser data, cryptocurrency wallets, and other valuable user information.

Open Source

Built with transparency in mind, NoMoreStealers is completely open source. You can inspect every line of code, contribute improvements, or adapt it for your specific needs. No hidden backdoors, no data collection - just pure protection.

Key Features

Anti-Spy Protection

Blocks screen capture and prevents unauthorized monitoring of your activities.

Process Protection

Monitors file system access and blocks malicious processes from accessing sensitive data.

Real-time Monitoring

Instant detection and blocking of stealer attempts with detailed event logging.
